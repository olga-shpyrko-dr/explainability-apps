Deploying the Generic Explainability App to DataRobot Codespaces

How to configure and launch the app in a customer DataRobot environment using a public GitHub repository.

Prerequisites

  • DataRobot account with Codespaces access enabled
  • DataRobot API token (from Developer Tools → API Keys)
  • A DataRobot deployment with SHAP explanations enabled (preferred), or a project + model ID
  • An AI Catalog scoring dataset ID (generate one using create_scoring_sample.py if needed)
  • GitHub account with access to the app repository
01

Clone the repository in your Codespace terminal

Once your Codespace is open, clone the repository and navigate into the app subfolder.

git clone https://github.com/olga-shpyrko-dr/explainability-apps.git
cd explainability-apps/generic-explainability

Keep .env out of version control — it is already in .gitignore. Domain config files (narrative_config.json, profile_config.json, feature_group_mapping.json) are safe to edit and commit for a specific customer deployment.

02

Obtain a scoring dataset (first-time only)

The app needs an AI Catalog dataset ID to score against. Use whichever path applies:

Option A — already generated locally: if you ran create_scoring_sample.py --upload on your machine, the dataset is already in the DR AI Catalog. Copy the printed ID directly into .env as SCORING_DATASET_ID. Skip to Step 3.

Option B — upload a CSV manually: drag-and-drop your scored CSV into DR AI Catalog via the UI, then copy the dataset ID into .env.

Option C — generate inside the Codespace: run the helper script after setting PROJECT_ID and MODEL_ID in .env.

cd backend
python create_scoring_sample.py \
  --upload \
  --dataset-name "Customer — Holdout Scoring Sample" \
  --use-case-id <DEFAULT_USE_CASE_ID>

The script prints the new dataset ID. Copy it into .env as SCORING_DATASET_ID.

03

Configure the app

Copy the template and fill in the values for this customer environment.

cd backend
cp .env.template .env
Variable Value
DATAROBOT_API_TOKEN Customer DR API token required
DATAROBOT_ENDPOINT https://app.eu.datarobot.com/api/v2 (adjust region) required
DEPLOYMENT_ID Deployment with SHAP enabled — preferred path recommended
PROJECT_ID + MODEL_ID Fallback if no deployment is available fallback
SCORING_DATASET_ID AI Catalog dataset to score (from Step 2) required
ROW_ID_COL Unique row identifier column name in the dataset required
OUTCOME_COL Binary 0/1 outcome column (enables actual rate in narrative) optional
DEFAULT_USE_CASE_ID Scopes the in-app dataset selector to one use case folder optional
DR_GATEWAY_MODEL azure/gpt-4o-mini — for AI narrative generation optional
APP_TITLE / APP_SUBTITLE Displayed in the app header optional
04

Customise domain configuration

Edit the three JSON files in backend/ to match the customer's use case. These can be committed back if you fork the repo.

narrative_config.json Entity labels, score labels, recommended action hint, and optional custom LLM prompt overrides.
profile_config.json Which columns appear in the cohort filter panel and individual profile view, plus score and explanation filter defaults.
feature_group_mapping.json Maps model feature names to named business domain groups shown in the Group Explanations chart.
05

Launch the app

Start the backend and frontend in two terminal tabs within the Codespace.

# Terminal 1 — FastAPI backend
cd backend
pip install -r ../requirements.txt
uvicorn main:app --host 0.0.0.0 --port 8000 --reload
# Terminal 2 — React frontend
cd frontend
npm install
npm run dev

DataRobot Codespaces will expose the forwarded ports and provide a shareable URL. Share the frontend port URL with the customer.

On first load the app runs the batch scoring pipeline against SCORING_DATASET_ID. This may take a minute depending on dataset size. Subsequent loads use the cached result.
Alternative: Deploy as a DataRobot Custom Application

For a persistent, shareable URL that lives inside DataRobot (not tied to a Codespace session), deploy the app as a Custom Application using the DataRobot App Framework. Steps 1–4 above (dataset, config, domain JSON) still apply — this section only covers the deployment step.

01

Install prerequisites (once per machine)

pip install datarobot-cli
curl -LsSf https://astral.sh/uv/install.sh | sh
dr auth set-url && dr auth login

dr auth set-url points the CLI at your DataRobot instance (e.g. https://app.eu.datarobot.com). dr auth login stores your API token.

02

Scaffold the App Framework base into the existing repo

The repo already has start-app.sh, backend/, and frontend/. The AF base layer adds the infrastructure manifest needed for dr run deploy — it will not overwrite existing files.

cd explainability-apps/generic-explainability
uvx copier copy https://github.com/datarobot/af-component-base .

When prompted, use explainability-app as the recipe name. Accept defaults for all other prompts.

03

Configure environment variables

Instead of manually editing .env, the AF CLI maps variables to DataRobot's secret management:

dr dotenv setup

Supply the same values as the .env table in Step 3 of the Codespace guide above (DATAROBOT_API_TOKEN, SCORING_DATASET_ID, DEPLOYMENT_ID, ROW_ID_COL, DR_GATEWAY_MODEL, etc.).

DR_GATEWAY_MODEL should be set to azure/gpt-5-1-2025-11-13 for the structured AI narrative to work correctly.
04

Deploy

dr run deploy

The CLI creates the Custom Application in DataRobot, builds the container, and outputs the shareable URL. The start-app.sh entry point (already in the repo) builds the React frontend and starts the FastAPI backend on port 8080 automatically.

On first load the app scores SCORING_DATASET_ID via batch prediction — this may take a minute. Subsequent loads use the cached result.
05

Redeploy after changes

Pull the latest code and redeploy in one command:

git pull
dr run deploy

To tear down the Custom Application:

dr run infra:down

Operational notes

  • The scoring cache is stored as backend/.prediction_dataset_cache.json. Delete it to force a fresh scoring run.
  • DEPLOYMENT_ID takes priority over PROJECT_ID + MODEL_ID — only set one path in .env to avoid ambiguity.
  • The AI narrative requires an LLM provider. Set DR_GATEWAY_MODEL=azure/gpt-5-1-2025-11-13 for DR SaaS environments. Alternatively configure DR_LLM_DEPLOYMENT_ID or a direct Anthropic/Azure key.
  • Keep .env out of git. Rotate the customer's API token after the demo if it was shared with you.
  • To switch the active dataset mid-session without restarting, use the dataset selector in the app's cohort panel — it calls POST /api/dataset/switch on the backend.
